## Directory Purposes

**`build/`:**
- Purpose: Static assets ready for production deployment
- Contains: Compiled HTML, CSS, images, PDFs
- Key files: `index.html` (entry point), `css/style.css` (compiled styles)
- Generated by: Tailwind CSS CLI during build process
- Committed: Yes (pre-built for static hosting)

**`src/`:**
- Purpose: Source files for preprocessing and build steps
- Contains: Raw CSS with Tailwind directives
- Key files: `input.css` (CSS source with custom animations)
- Generated: No
- Committed: Yes (source of truth for styling)

**`.planning/`:**
- Purpose: GSD analysis and planning documentation
- Contains: Architecture, structure, conventions, concerns analysis
- Generated: By GSD mapper tool
- Committed: Yes (reference for future development)

**`build/images/icons/`:**
- Purpose: SVG assets organized by technology category
- Contains: `backend/` (databases, servers), `design/` (Figma, Photoshop), `frontend/` (React, Vue)
- Naming: Lowercase technology-name.svg format (e.g., `javascript-plain.svg`)

## Naming Conventions

**Files:**
- HTML: `index.html` (single entry point)
- CSS: `input.css` (source), `style.css` (compiled output)
- Configuration: Lowercase with extensions (`.js`, `.yaml`, `.json`)
- Documentation: Uppercase.md format (README.md, ARCHITECTURE.md)
- Assets: Lowercase with hyphens (e.g., `Shafi Uzman Fassy - Resume.pdf`)
- Icons: `{technology}-{variant}.svg` format (e.g., `javascript-plain.svg`)

**Directories:**
- Lowercase (src, build, images)
- Plural for collections (icons, images, docs)
- Semantic names reflecting purpose (backend, frontend, design)

**HTML Elements:**
- IDs (used for navigation): lowercase, hyphenated (`id="home"`, `id="about"`, `id="contact"`)
- Classes (Tailwind utilities): kebab-case Tailwind prefixed (`md:text-4xl`, `hover:text-indigo-500`)
- Section elements: Self-closing nav, semantic section tags for each content area

## Where to Add New Code

**New Feature (JavaScript):**
- Location: Add new `<script>` block at end of `build/index.html` before closing `</body>`
- Pattern: Follow vanilla JavaScript with DOM API; declare functions or classes at module level
- Example: New scroll animation would add TextScramble-like class and initialize it

**New Section (Content):**
- Location: Add new `<section>` element in `build/index.html` body with unique id attribute
- Pattern: Follow existing section pattern with id for anchor linking, Tailwind utilities for styling
- Update navbar: Add new `<li><a href="#new-section">Label</a></li>` to navbar nav element
- Update Tailwind config: If using new color/animation, extend theme in `tailwind.config.js`

**New Styling (CSS):**
- Location: Add to `src/input.css` for custom rules, use Tailwind utility classes in HTML directly
- Pattern: Use Tailwind @directives for custom utilities, CSS custom properties for theming
- Compile: Run Tailwind CLI to generate `build/css/style.css`

**New Utilities/Helpers (JavaScript):**
- Location: New `<script>` block at end of `build/index.html` or extract to separate file in build/
- Pattern: Declare as function or class, initialize from separate script block
- Scope: Currently global scope (window object); consider namespace if multiple utilities

**New Assets (Images/Icons):**
- Location: `build/images/` (icons: `build/images/icons/{category}/`, projects: `build/images/project banners/`)
- Pattern: SVG format for icons (scalable, small filesize), PNG/JPG for project banners
- Update HTML: Reference in img src with relative path (e.g., `images/icons/backend/technology.svg`)

## Special Directories

**`build/`:**
- Purpose: Production deployment target
- Generated: Yes (by Tailwind CSS CLI)
- Committed: Yes (pre-compiled for static hosting)
- Notes: Directory is served directly by Render.com as static site root

**`.git/`:**
- Purpose: Version control metadata
- Generated: Yes (by git init)
- Committed: No (system directory)

**`node_modules/`:**
- Purpose: Installed npm packages (Tailwind CSS)
- Generated: Yes (by npm install)
- Committed: No (ignored in .gitignore)

## Build Process

**Tailwind CSS Compilation:**
- Input: `src/input.css` (CSS source with @tailwind directives)
- Output: `build/css/style.css` (compiled with all Tailwind utilities + custom styles)
- Command: Defined in package.json scripts or Render build hook
- Dependencies: Node.js, npm, Tailwind CLI

**Deployment:**
- Hosting: Render.com static site service
- Config: `render.yaml` defines build command and static publish path
- Trigger: Git push to main branch
- Caching: Headers configured for assets (images: 1 year, docs: 1 year, others: 1 year)
